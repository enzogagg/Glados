# Usage Guide

CLaD is a compiled language that targets bytecode, which is then interpreted by a virtual machine. The workflow therefore consists of two steps.

## 1. Write Code

Create a source file with the `.clad` extension.

**Example: `hello.clad`**
```clad
principal
    afficher("Bonjour CLaD !")
fin
```

## 2. Compile (`glados-compiler`)

Use the compiler to transform your source code into a binary file (`.cbc`).

**Syntax:**
```bash
./glados-compiler [source_file]
```

**Example:**
```bash
./glados-compiler hello.clad
```
This will generate an `out.cbc` file (or the specified name depending on options).

## 3. Execute (`glados-vm`)

Run the virtual machine passing the generated bytecode file.

**Syntax:**
```bash
./glados-vm [bytecode_file]
```

**Example:**
```bash
./glados-vm out.cbc
```
> Output: `Bonjour CLaD !`

## Workflow Summary

```bash
# 1. Edit
nano main.clad

# 2. Compile
./glados-compiler main.clad

# 3. Execute
./glados-vm out.cbc
```

## Debugging Options

If you encounter issues, the VM can often provide details about the error (empty stack, unknown instruction...). Make sure to read the standard error messages.
